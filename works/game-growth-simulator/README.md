# ゲーム育成シミュレーター（静的攻略ツール）

ゲームの育成要素を分析するために作成した  
**完全クライアントサイド型の Web シミュレーター**です。

ゲームプレイを開始する前段階で、  
「どの行動を、どの順序で選択するか」を検証・整理するための  
**計算簡易化ツール**として開発しました。

※ ゲーム素材を使用しているため、公開リンクは設けていません。

---

## 概要

本ツールでは、  
育成フェーズ全体（序盤〜終盤）について、

- 各ターンの行動内容
- 装備や強化段階
- ステータスのランク推移
- 特殊進行要素の達成状況
- 消費アイテムの使用有無

をすべて入力・保存し、  
**1 ターン目から最終ターンまでを通しでシミュレーション**できます。

設定内容とシミュレーション結果は  
ローカルストレージおよび JSON として管理され、  
ブラウザのみで即座に利用可能です。

---

## 主な機能

- 全ターン（最大 60 ターン超）の時系列シミュレーション
- 行動種別ごとの基礎値・人数（期待値）反映切り替え
- 構成情報からの **期待人数・効果量の計算**
- 確率分布に基づく **期待値計算（下振れカット対応）**
- 特殊効果計算のオン／オフ切り替え
- JSON インポート／エクスポートによる再利用
- 完全オフライン動作（HTML 単体）

---

## 設計方針

### 「使えるようになるまでの速さ」を最優先

本ツールは、

- 友人間での検証・攻略用途
- 使用期間は長くても数か月
- ゲームアップデート直後から使えることが重要

という前提があったため、

- サーバーを持たない
- ビルド・デプロイ不要
- ファイルを開くだけで動作

という **完全静的構成**を採用しています。

---

## 実装の特徴（ソースベース）

### 1. ターン・状態管理の一元化

各ターンは  
「フェーズ・進行月・前半／後半」といった要素を組み合わせた  
**一意なスロットキー**で管理しています。

- 全スロットを初期化時に生成
- 保存データが存在しない場合でもデフォルト値を補完
- Map 構造を用いて高速に参照・更新

これにより、  
途中のターンだけ編集しても全体が破綻しない構成になっています。

---

### 2. 行動定義をデータとして分離

行動内容は、以下の情報を持つデータ定義として管理しています。

- 行動種別
- 表示ラベル
- 基礎獲得量
- 人数（期待値）を反映するかどうか

この定義を元に、

- UI 表示
- 計算ロジック
- 保存形式

を共通化しており、  
後から行動を追加・調整しやすい構造です。

---

### 3. 期待値計算（DP ベース）

構成情報からの期待値計算では、

- 各要素の発生確率を重みとして保持
- 状態を集約しながら遷移させる
- 同一状態をまとめることで計算量を抑制

という **動的計画法（DP）ベースの分布計算**を行っています。

その結果から、

- 属性別の期待値
- 上位要素の加重平均
- 上振れ％を考慮した下振れカット平均

を算出しています。

---

### 4. 特殊効果計算の割り切り実装

一部の特殊効果については、

- 全分岐を厳密に列挙するのではなく
- 重み付けと残容量を用いた **期待値ベースの近似**

で処理しています。

精度と計算速度のバランスを優先し、  
検証用途として十分な再現性を確保しています。

---

### 5. ローカル完結のデータ管理

- 設定・シミュレーション結果は localStorage に保存
- JSON 形式でまとめて入出力可能
- バージョン番号を保持し、将来の互換性にも配慮

これにより、

- ブラウザを閉じても続きから再開
- 友人間での検証データ共有

が容易になっています。

---

## 使用技術

- HTML / CSS
- Vanilla JavaScript
- localStorage
- JSON

外部フレームワークやライブラリは使用していません。

---

## 成果・効果

- 育成方針の検討・検証時間を大幅に短縮
- アップデート初日から即利用可能
- オフライン配布・共有が容易
- 「短期間だけ使う攻略ツール」に適した設計判断を実践

---

## このツールで意識したこと

- 長期運用を前提にしない割り切り
- 正確さより「検証に十分な再現性」
- 作ることより「早く使える」ことを優先

本ツールは、  
プロダクトとして育て続けるものではなく、  
**攻略・検証という一時的な需要に最大効率で応える**ためのツールです。
