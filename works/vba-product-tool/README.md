# Excel VBA シリンダーデータ登録・検索ツール

社内で運用されていたシリンダー関連データを、  
「Excel直打ち・備考欄運用」から脱却させるために開発した  
**Excel VBA製のユーザーフォーム型 業務改善ツール**です。

---

## 概要

従来は以下のような課題がありました。

- 同じ情報を何度も手入力している
- データ形式が人によってバラつく
- 検索や流用がしづらい
- シート構造変更でマクロが壊れやすい

これらを解決するため、  
**登録・検索をUI操作だけで行えるデータベース的ツール**として設計しています。

---

## 主な機能

- シリンダーデータの登録・検索（ユーザーフォーム）
- 受注データからの自動反映
- マスタ参照による候補選択
- 半自動入力（反映モード）
- 検索結果の安全な外出し表示
- 落版予定管理（色目別シリンダー操作）

---

## 使用技術

- Excel VBA
- UserForm / ListBox / ComboBox
- Win32 API（マウスホイール対応）
- Dictionary / Collection

---

## 実装上の工夫

- 列番号に依存しない **見出し名ベースのマッピング設計**
- 入力バリデーション・正規化処理を共通モジュール化
- 受注データとシリンダーデータの連動更新
- 色目（0～9色）ごとの動的UI制御
- マウスホイール・解像度スケーリング対応
- 防御的エラーハンドリングによる長期運用耐性

---

## ChatGPT の活用について

本プロジェクトでは、実装を自動生成させる目的ではなく、  
**設計整理・処理分割・確認用レビュー補助**として ChatGPT を活用しました。

### 活用内容

- 業務フローを文章化し、処理単位に分解するための壁打ち
- UserForm イベントが多段に連動する構造の整理
- 検索／表示／落版／履歴参照といった責務分離の明確化
- VBAコードを第三者視点で読み直すための確認用レビュー
- 仕様追加・変更時の影響範囲の洗い出し

### 実装に対する姿勢

- 実際のコードはすべて自分で読み、理解し、編集しています
- 既存処理を変更する際は、必ず影響範囲を確認してから修正しています
- ChatGPT の出力はそのまま採用せず、  
  既存コード・業務仕様と一致するかを必ず照合しています

### ChatGPT を使う理由

Excel VBA のような **状態管理が複雑でデバッグしづらい環境**では、  
「処理構造を言語化できるかどうか」が品質に直結します。

ChatGPT は  
- 実装スピードを上げるためのツールではなく  
- **設計を破綻させないための補助ツール**  
として利用しています。

---

## 成果・効果

- 再入力・入力ミスを大幅削減
- 現場で長期利用される業務ツールとして定着
- Excel業務でも壊れにくい設計思想を実践
- 機能追加・仕様変更に対応しやすい構造を維持
- 現在も運用しながら改善・拡張を継続中
