# ゲーム用ステータス可視化・計算ツール

スプレッドシートで管理されていたゲーム内数値を、  
「手動計算・静的グラフ」中心の運用から脱却させるために  
開発・改善を行った **Web ベースのステータス可視化・計算ツール**です。

---

## 概要

開発以前は、以下のような運用が行われていました。

- スプレッドシートで数値を管理
- グラフを画像として書き出し、ブログや SNS に貼り付けて利用
- 条件変更のたびに再計算・再作成が必要
- 複数条件の比較や試行錯誤がしづらい

これらの課題を解消するため、  
**条件を選択するだけで数値計算とグラフ表示を行えるツール**として  
設計・改善されています。

---

## 開発体制と立ち位置

本ツールの **初期ベース実装は、知り合った上級エンジニアが担当**しています。

私は、

- 開発初期から仕様・構成の相談に参加
- 既存コードを理解した上での機能追加・改善
- 数値計算ロジックの一部実装・調整
- リリース後の UI 改善・挙動修正・データ更新対応

という形で関わりました。

特にリリース後は、  
**実際の利用状況を見ながらの改善・保守を一貫して担当**しています。

---

## 主な機能

- ステータス条件の選択による数値計算
- 複数条件の同時比較
- グラフによる成長・性能の可視化
- カード・キャラクター情報の選択 UI
- 設定内容に応じた自動再計算

---

## 使用技術

- React（関数コンポーネント / Hooks）
- TypeScript（null / undefined を意識した型設計）
- 連動セレクトを含む状態管理
- CSV ベースのデータ駆動設計
- 既存コードを壊さない段階的リファクタリング

※ 本プロジェクトが  
  **初めての Web ツール開発かつ、初めての本格的な React 利用**  
※ それ以前の開発経験は、主に Excel / VBA が中心  
※ 本プロジェクトを通して、  
  DOM 構造・状態管理・イベント設計といった  
  Web フロントエンドの基本概念を実践的に習得  
※ 現在は、HTML / CSS / JavaScript ベースの実装であれば  
  構造を意識した設計・実装が自然に行えるようになった

## 実装上の主な改善点

### 選択 UI の再設計（キャラ名 → カード名）

初期設計では、

- キャラクター名で絞り込み
- その後カードを選択

という導線を前提としていました。

しかし、リリース後の利用状況から、

- カード名の方がユーザーに認知されている
- カード画像から探したいという要望が多い

ことが分かり、  
**既存仕様を壊さずに選択の自由度を上げる改善**を行いました。

#### 対応内容

- 未選択状態ではカード一覧をすべて表示
- カード名が 1 つに絞られた場合は自動選択
- 画像付きプルダウンによる視認性向上
- 選択順を固定しない設計への変更

UI・状態管理・データ取得ロジックが  
複数ファイルにまたがっていたため、  
影響範囲を確認しながら段階的に修正しています。

---

### 未選択状態の扱い整理（null → undefined）

連動セレクトの挙動を安定させるため、

- 候補リストに null を混ぜる設計を廃止
- 未選択は value を持たない状態（undefined）で統一

という方針に変更しました。

これにより、

- セレクトの判定ズレ
- 表示・自動選択ロジックの不整合

を解消しています。

---

### 自動補完・自動選択ロジックの追加

カード名を選択した際に、

- 他の項目が一意に定まる場合
- かつ、未選択の項目がある場合

は、  
**関連項目を自動で補完する挙動**を追加しました。

- 未選択時のみ自動セット
- 明示的に選択されている場合は上書きしない

といった条件を設け、  
操作性と状態の一貫性の両立を意識しています。

---

### 計算ロジックの調整と安定化

当初は、  
私が **再現度を重視した詳細な計算ロジック**を担当していました。

しかし、長期運用と引き継ぎを見据え、

- 再現度を適切に調整
- アップデートごとに細かく追従しなくても
  結果の差分が出にくい計算方式へ変更

する判断を行いました。

理論上の正確さよりも、

- 条件比較に使える一貫性
- 運用・保守のしやすさ

を優先した設計へ整理しています。

---

### データ構造の拡張と再設計

運用フェーズを考慮し、

- 初期条件を表す数値項目
- 段階的に変化する効果情報
- 特定条件・環境に依存する補正要素

を扱えるよう、  
データ定義の見直しと拡張を行いました。

---

### 引き継ぎ対応と外部運用支援

個人的にゲームを離れるタイミングで、  
本ツールの **メイン処理・運用を知り合いの YouTuber に引き継ぎ**ました。

引き継ぎにあたっては、

- データ更新方法の説明
- アップデート時の計算ロジック調整方針の共有
- YouTuber 側の Cloudflare 環境でのデプロイ・運用支援

を行い、  
**非エンジニアでも継続運用できる状態**を整えています。

## 実装上で特に苦戦した点・進め方

本プロジェクトは、  
**Web ツール開発自体が初めての経験**であり、  
それ以前は Excel / VBA を中心に開発を行っていました。

そのため、

- React の状態管理
- UI 変更が計算ロジックに与える影響
- 再現度・使いやすさ・保守性のトレードオフ判断

に特に苦戦しました。

実際には、

- 利用者からの質問
- 想定外の使われ方
- 数値への違和感の指摘

を元に、

- 仮説を立てる
- 小さく修正する
- 影響範囲を確認する

というサイクルで改善を進めています。
