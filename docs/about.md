# About

業務改善・自動化を中心に、身の回りの不便さをどうにかするためのツールを作っています。業務・個人を問わず、「これ、作った方が早くないか？」と思ったものを形にして試す、ということを続けてきました。特別なきっかけがあったというより、手作業が多いところや分かりにくい仕組みに触れるたびに手を動かしているうちに、自然とこうした開発を続けるようになっていました。

## 開発の原点

昔から、ゲームを遊ぶ際もプレイそのものだけでなく、どうすれば効率が良くなるか、どの選択が有利かといったことを考え、データを集めて眺めたり比較したりすることを楽しんでいました。Excel を使うようになってからは、関数を使って汎用的な計算ツールや検証用の表を作るようになり、「1つでいろいろなケースをカバーできる」ことに面白さを感じていました。プログラムを書けばさらに自由度が上がり、「こうしたらもっと良くなるのでは？」という考えをそのまま形にできます。狙った通りの結果が出たときの感覚が楽しく、うまくいかなければ直し、使いにくければ作り直すという試行錯誤を自然に続けています。

## 業務改善・ツール開発について

業務改善では、完全な自動化よりも、安全に任せられること、長く使い続けられることを重視しています。Excel VBA、Python、JavaScript などを使い、入力のばらつきや属人化を減らすためのツールや、人が目で見て判断していた作業をロジックとして整理する仕組みを作ってきました。とりあえず動くものを作るのではなく、実際の運用やデータ量を想定しながら、壊れにくさや修正しやすさを意識して設計しています。

## 表示・操作への意識

ツールを作る中で、処理そのものだけでなく「どう見せるか」「どう操作してもらうか」が、使われ続けるかどうかに大きく影響することを強く意識するようになりました。ロジックが正しくても、表示が分かりにくかったり操作に迷うものは触ってもらえません。Excel やスプレッドシートで検証ツールを作っていた頃、中身は正しくてもシートのままだと使われなくなる経験を何度もしました。そこから、HTML や Python の画面化、VBA のユーザーフォームなど、「触りやすい形」にすることの重要性を学びました。同じ処理でも、スプレッドシートと Web ページではユーザー視点がまったく違う、その違いを意識しながら表示や操作を設計することを大切にしています。

## 開発スタンス

新しい技術そのものも好きですが、目的は技術を使うことではなく、再現できることを増やすことだと考えています。目の前の問題をどう整理し、どう形にすれば良くなるかを考えている時間が、自分にとって一番のモチベーションです。業務改善ツール、自動化スクリプト、Bot、シミュレーターなどジャンルはさまざまですが、「作って試して、直して、また良くする」という流れは一貫しています。

## このポートフォリオについて

このポートフォリオには、業務改善ツール、自動化スクリプト、Bot、シミュレーターなどをまとめています。それぞれのプロジェクトでは、背景や設計意図、工夫した点が分かるように README を用意しています。実際に使われること、使い続けられることを前提にした取り組みとして見ていただければ嬉しいです。
